## 예산 설계
- **월별 예산**: 비용 통계 화면(cost-analytics.html)에서 클래스별 예산 게이지를 설정하고 사용률을 확인합니다.
- **모델별 한도**: GPT/Claude/Gemini 등 모델별 일일/월간 호출 수 또는 비용 상한을 지정합니다.
- **정산 주기**: 수익/정산 관리(revenue-settlement.html) 화면 기준으로 월 단위 정산일·정산 상태(정상/대기/완료)를 운영자가 관리합니다.

## 가드레일 정책
1. **경고 임계치**: 예산 80% 도달 시 슬랙/이메일 알림을 발송합니다.
2. **하드 리밋**: 100% 초과 시 고비용 모델 호출을 제한하고, 대체 모델 제안을 노출합니다.
3. **클래스 차단**: 특정 클래스가 예산을 초과하면 과제/실습 제출을 일시중단하고 운영자 승인을 요구합니다.
4. **정산 보류 처리**: 정산 상태가 "대기"인 프로젝트는 추가 비용 발생을 제한하거나 경고 배너를 표시합니다.

## 모니터링 포인트
- **차트/지표**: 시간대별 호출, 일별 비용 추세, 모델 사용 비율 변화를 모니터링해 급증 구간을 조기에 발견합니다.
- **알림 로그**: 예산 임계치 알림과 차단 이벤트를 감사 로그에 저장하고 내보내기(이메일/CSV)를 지원합니다.
- **예외 관리**: 시험/해커톤 등 일시적 트래픽 증가 시 임시 상한선을 조정하고, 종료 후 원복합니다.

## 운영 체크리스트
- 새 클래스 개설 시 기본 예산·상한·알림 채널을 즉시 설정합니다.
- 정산 상태가 "대기"인 프로젝트를 우선 확인해 비용 발생을 제한하거나 확인 알림을 띄웁니다.
- 모델 최적화 제안(예: Gemini Pro 전환)을 비용 대시보드 제안 카드에 노출해 사용을 유도합니다.
