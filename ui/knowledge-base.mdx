## 화면 한눈에 보기

- **헤더/서브내비**: 클래스 선택 드롭다운과 최근 대화, 설정, 로그아웃 진입점이 사이드바에 배치되어 있습니다.
- **모드 선택 영역**: "간편 업로드"(자동 청킹/검색 파라미터)와 "고급 설정"(URL 크롤링·세부 파라미터)을 카드로 전환합니다. 선택 시 하단 단계 카드가 해당 모드에 맞게 바뀝니다.
- **지식베이스 생성 단계**: 업로드 → 설정 확인 → 생성 3단계 카드가 나란히 노출되며, 필요 시 "지식베이스로 돌아가기" 버튼으로 목록으로 복귀합니다.
- **목록/미리보기 패널**: 좌측은 문서 목록·필터·상태(인제스트 중/완료)를, 우측은 제목·섹션·출처 링크까지 포함한 프리뷰를 제공합니다.
- **비교/테스트 영역**: 두 개의 RAG 설정을 나란히 구성해 검색/임베딩 모델, top-k/threshold, 재랭크 on/off를 바꿔가며 응답과 출처를 비교합니다.

## 주요 워크플로우

### 1) 지식베이스 생성
1. **방식 선택**: 모드 카드에서 간편 업로드 또는 고급 설정을 선택합니다.
2. **파일 업로드**: PDF/문서 파일을 드래그 앤 드롭하거나 파일 선택 버튼을 사용합니다.
3. **설정 검토**:
   - 간편 업로드: 기본 청킹/검색 파라미터가 자동 적용됩니다.
   - 고급 설정: URL 크롤링(도메인 제한/심층 크롤링)과 청킹/임베딩/검색 파라미터를 직접 입력합니다.
4. **생성**: "지식베이스 생성" 버튼으로 인제스트를 시작하고, 목록에서 진행 상태를 확인합니다.

### 2) 문서 상태 확인 및 탐색
- **필터링**: 검색창과 상태/파일 유형 필터로 특정 자료를 빠르게 찾습니다.
- **상세 보기**: 문서 카드 클릭 시 우측 프리뷰에서 섹션 계층, 하이라이트, 인덱싱된 출처를 확인합니다.
- **버전 관리**: 업데이트된 파일은 "최신/과거 버전" 토글로 비교합니다.

### 3) RAG 비교 테스트
1. 좌우 **설정 카드**에서 임베딩/검색/재랭크 파라미터를 각각 입력합니다.
2. **테스트 프롬프트**를 동일하게 입력하고 실행하면 응답과 출처가 나란히 표시됩니다.
3. 응답 품질뿐 아니라 **속도/토큰 사용량**도 비교하여 운영 기준을 정리합니다.

## 권장 설정 팁
- **초기 실습**: 간편 업로드로 시작해 기본 파라미터(자동 청킹, 최적화된 검색) 품질을 확인한 뒤, 고급 설정에서 필요한 값만 조정합니다.
- **재랭킹 사용**: 정확도를 우선하는 수업 자료는 재랭킹을 켜고, **TTFT/latency**가 민감한 경우 off 상태에서 top-k를 줄여 테스트합니다.
- **출처 노출**: 프리뷰와 테스트 결과 모두에서 출처 링크를 확인할 수 있으므로, 학습용 KB는 문서별 출처 메타데이터를 필수로 포함시킵니다.
